# 价格管理页面实现计划

## 1. 需求概述

创建一个专门的价格管理页面，用于管理和查看串珠、手串配件和手串成品的价格变更历史，支持以下功能：
- 商品价格变更记录
- 价格走势图表展示
- 按商品类型筛选
- 时间范围筛选

## 2. 技术方案

### 2.1 后端实现（Django）

#### 2.1.1 数据模型
在 `Server/store/models.py` 中新增 `PriceHistory` 模型：
- 关联 Product 外键
- 记录变更前价格
- 记录变更后价格
- 记录价格类型（采购成本/成本价格/销售价格）
- 记录操作人
- 记录变更时间

#### 2.1.2 数据库迁移
创建并应用数据库迁移文件

#### 2.1.3 修改商品更新逻辑
在 `update_product` 视图中，当价格变更时自动记录历史

#### 2.1.4 新增 API 端点
- 获取价格历史列表（支持筛选）
- 获取价格图表数据

### 2.2 前端实现（Vue 3 + Element Plus）

#### 2.2.1 API 封装
在 `Page/src/api/store.js` 中添加价格历史相关 API 函数

#### 2.2.2 页面组件
创建 `Page/src/views/PriceManagement.vue` 组件：
- 商品筛选器（类型、名称、时间范围）
- 价格历史表格
- 价格走势图表（使用 ECharts）

#### 2.2.3 路由配置
在 `Page/src/router/index.js` 中添加价格管理页面路由

#### 2.2.4 菜单配置
在 `Page/src/layout/index.vue` 中添加菜单项

## 3. 详细实施步骤

### 后端步骤
1. 创建 PriceHistory 模型
2. 生成并应用迁移文件
3. 修改 update_product 视图以记录价格变更
4. 添加新的 API 视图函数
5. 更新 URL 路由

### 前端步骤
1. 安装 ECharts 依赖
2. 添加 API 封装函数
3. 创建价格管理页面组件
4. 配置路由和菜单

## 4. 预期文件变更

| 文件路径 | 变更类型 | 说明 |
|---------|---------|------|
| Server/store/models.py | 修改 | 添加 PriceHistory 模型 |
| Server/store/migrations/... | 新增 | 数据库迁移文件 |
| Server/store/views.py | 修改 | 添加价格历史 API，修改商品更新逻辑 |

| Page/src/router/index.js | 修改 | 添加价格管理路由 |

## 5. 技术要点

- 使用 ECharts 实现图表展示
- 价格变更记录要支持多种价格类型
- 响应式设计，支持移动端显示
- 保留现有代码风格和架构
