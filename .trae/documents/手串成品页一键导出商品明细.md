# 手串成品页一键导出商品明细

## 概述

在手串成品页添加"导出明细"按钮，一键导出所有手串成品的完整明细信息为 Excel (.xlsx) 文件，包含 3 个 Sheet：成品总览、串珠明细、配件明细。

## 当前状态分析

### 前端
- **父组件** `Products.vue`：含 3 个 Tab（手串成品/串珠/手串配件），头部 `header-actions` 区域（第 7-15 行）有"批量删除"和"添加商品"按钮
- **子组件** `ProductTable.vue`：展示商品列表，展开行显示成品串珠和配件明细
- **API 封装** `store.js`：`getProducts(params)` 调用 `GET /products/`，支持分页参数 `page` / `page_size`，返回 `{ products, total_count, page, page_size }`
- **数据加载** `fetchProducts()`（Products.vue 第 1022 行）：按当前 Tab 类型筛选，分页获取，默认 `page_size=50`
- **已安装依赖**：vue、element-plus、axios、echarts、pinia、vue-router。**无任何导出库**

### 后端
- **API 视图** `get_products`（views.py 第 1015-1182 行）：分页返回商品列表，成品类型已包含完整 `finished.beads[]` 和 `finished.accessories[]` 明细数据
- **SKU 序列化** `sku_to_dict`（views.py 第 52-72 行）：返回 sku_code、name、material、size、color、purchase_cost、cost_price、weight、quality_level、selling_price 等
- **无导出相关依赖**（无 openpyxl/pandas）

### 成品明细数据结构（API 已返回）
```
product = {
  id, code, name, product_type, cost_price, selling_price,
  location, supplier, is_active, created_at,
  finished: {
    beads: [{ bead_code, bead_name, sku: {name, sku_code, ...}, quantity, bead_cost_price, bead_purchase_cost, bead_size, bead_quality_level }],
    accessories: [{ accessory_code, accessory_name, sku: {name, sku_code, ...}, quantity, accessory_cost_price }],
    labor_cost, elastic_cost
  }
}
```

## 实现方案

采用**纯前端导出**方案：安装 `xlsx` + `file-saver` 库，前端调用现有 `getProducts` API 获取全量数据（通过设置大 `page_size` 或循环分页），在浏览器中生成 Excel 文件。

### 步骤 1：安装前端依赖

在 `d:\workplace_shop\WeWin\Page` 目录执行：
```bash
npm install xlsx file-saver
```

### 步骤 2：新增导出工具模块

**新建文件** `d:\workplace_shop\WeWin\Page\src\utils\exportProducts.js`

功能：接收成品商品数组，生成含 3 个 Sheet 的 Excel 工作簿并触发下载。

```javascript
import * as XLSX from 'xlsx'
import { saveAs } from 'file-saver'

export function exportFinishedProducts(products) {
  const wb = XLSX.utils.book_new()

  // Sheet 1: 成品总览
  const overviewData = products.map(p => ({
    '货号': p.code,
    '商品名称': p.name,
    '成本价格': p.cost_price,
    '售卖价格': p.selling_price,
    '利润率(%)': p.selling_price > 0 ? ((p.selling_price - p.cost_price) / p.selling_price * 100).toFixed(2) : '0',
    '工费': p.finished?.labor_cost ?? 0,
    '弹性成本': p.finished?.elastic_cost ?? 0,
    '串珠数量': p.finished?.beads?.length ?? 0,
    '配件数量': p.finished?.accessories?.length ?? 0,
    '库位': p.location || '',
    '供应商': p.supplier || '',
    '状态': p.is_active ? '启用' : '禁用',
    '创建时间': p.created_at ? new Date(p.created_at).toLocaleString() : ''
  }))
  const ws1 = XLSX.utils.json_to_sheet(overviewData)
  XLSX.utils.book_append_sheet(wb, ws1, '成品总览')

  // Sheet 2: 串珠明细
  const beadsData = []
  products.forEach(p => {
    (p.finished?.beads || []).forEach(b => {
      beadsData.push({
        '成品货号': p.code,
        '成品名称': p.name,
        '串珠货号': b.bead_code,
        'SKU名称': b.sku?.name || b.sku?.sku_name || b.bead_name,
        '数量': b.quantity,
        '克价(元/克)': b.bead_purchase_cost ?? '',
        '单价': b.bead_cost_price,
        '小计': (b.bead_cost_price * b.quantity).toFixed(2),
        '规格(mm)': b.bead_size || '',
        '品级': b.bead_quality_level || ''
      })
    })
  })
  const ws2 = XLSX.utils.json_to_sheet(beadsData)
  XLSX.utils.book_append_sheet(wb, ws2, '串珠明细')

  // Sheet 3: 配件明细
  const accessoriesData = []
  products.forEach(p => {
    (p.finished?.accessories || []).forEach(a => {
      accessoriesData.push({
        '成品货号': p.code,
        '成品名称': p.name,
        '配件货号': a.accessory_code,
        'SKU名称': a.sku?.name || a.sku?.sku_name || a.accessory_name,
        '数量': a.quantity,
        '单价': a.accessory_cost_price,
        '小计': (a.accessory_cost_price * a.quantity).toFixed(2)
      })
    })
  })
  const ws3 = XLSX.utils.json_to_sheet(accessoriesData)
  XLSX.utils.book_append_sheet(wb, ws3, '配件明细')

  // 生成并下载
  const wbout = XLSX.write(wb, { bookType: 'xlsx', type: 'array' })
  const blob = new Blob([wbout], { type: 'application/octet-stream' })
  const timestamp = new Date().toISOString().slice(0, 10)
  saveAs(blob, `手串成品明细_${timestamp}.xlsx`)
}
```

```javascript
const handleExportProducts = async () => {
  exportLoading.value = true
  try {
    // 获取全量成品数据（不分页）
    const allProducts = []
    let page = 1
    const pageSize = 200
    while (true) {
      const response = await getProducts({
        product_type: 'finished',
        page,
        page_size: pageSize
      })
      allProducts.push(...response.data.products)
      if (allProducts.length >= response.data.total_count) break
      page++
    }
    if (allProducts.length === 0) {
      ElMessage.warning('没有可导出的成品数据')
      return
    }
    exportFinishedProducts(allProducts)
    ElMessage.success(`成功导出 ${allProducts.length} 个成品明细`)
  } catch (error) {
    console.error('导出失败:', error)
    ElMessage.error('导出失败，请重试')
  } finally {
    exportLoading.value = false
  }
}
```

## 假设与决策

1. **纯前端导出**：无需修改后端，复用现有 `getProducts` API，降低改动风险
2. **全量获取策略**：循环分页请求（每页 200 条），直到获取所有成品数据，确保导出完整
3. **3 个 Sheet 结构**：成品总览（汇总信息）+ 串珠明细（每个串珠一行，关联成品）+ 配件明细（每个配件一行，关联成品），通过"成品货号"列关联
4. **仅手串成品 Tab 显示导出按钮**：用户明确要求是手串成品页的功能
5. **文件名含日期**：`手串成品明细_YYYY-MM-DD.xlsx`，便于管理多个导出文件
6. **利润率计算**：`(售价 - 成本) / 售价 * 100`，与页面展示逻辑一致
